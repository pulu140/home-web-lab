# home-web-lab

## 项目简介

`home-web-lab` 是一个用于归档和发布多个独立 HTML 页面的仓库。它不针对任何特定域名或具体网站，主要用于存放各类前端页面、实验性网页、静态组件或小型 Web 示例。你可以将此仓库视为一个个人或团队的前端页面实验室，用于保存、共享和迭代各种 HTML 资源。

## 目录说明

仓库根目录下包含以下主要内容：

- `index.html`（可选）：仓库首页或导航页面，用于链接各子页面。
- `pages/`：存放主要 HTML 页面文件的目录。每个页面可以是一个独立的功能模块、实验作品或静态页面。
- `assets/`：存放页面所需的静态资源，如 CSS 样式表、JavaScript 脚本、图片、字体等。
- `docs/`（可选）：存放与页面相关的文档、说明或使用指南。
- `examples/`（可选）：存放更小的示例代码或演示片段。

> 实际目录结构可能随仓库内容更新而变化，请以仓库当前文件为准。

## 页面归档说明

本仓库中的 HTML 页面遵循以下归档原则：

- **独立性**：每个页面尽可能自包含，减少对外部资源的依赖。公共资源（如共用样式、脚本）可集中存放于 `assets/` 目录。
- **命名规范**：页面文件名应清晰反映其内容或用途，例如 `clock.html`、`todo-list.html`、`canvas-demo.html`。
- **版本管理**：页面更新通过 Git 提交记录追踪。如需保留历史版本，建议在提交信息中注明改动内容。
- **兼容性**：页面不强制要求跨浏览器完全一致，但应尽量保持基本功能可用。实验性页面可在文件开头注释中注明测试环境。

## 维护说明

- **添加新页面**：将 HTML 文件放入 `pages/` 目录，并确保引用资源路径正确。如需在首页显示链接，请更新 `index.html`。
- **更新已有页面**：直接修改文件并提交。若页面结构发生重大变化，建议在提交信息中说明。
- **删除页面**：移除对应文件即可。如该页面在 `index.html` 中有链接，请同步删除。
- **资源管理**：请勿在仓库中存放过大文件（如超过 100MB 的媒体文件）。建议使用外部链接或压缩资源。
